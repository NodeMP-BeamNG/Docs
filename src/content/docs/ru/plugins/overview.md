NodeMP выполняет **серверные Lua-плагины**, которые реагируют на происходящее на вашем
сервере: подключение игроков, спавн автомобилей, сообщения чата, консольные команды, таймеры и
многое другое. С помощью плагинов вы добавляете команды администрирования, сохранение
состояния, пользовательские игровые режимы, MOTD — всё, что выходит за рамки базовой
ретрансляции.

Поскольку движок плагинов портирован из BeamMP, **серверные плагины BeamMP работают на NodeMP
практически без изменений**, и вы дополнительно получаете более чистый собственный API
`NodeMP.*` от NodeMP. Подробности совместимости см. в
[Совместимости с BeamMP](/ru/introduction/beammp-compatibility/).

## Где располагаются плагины

Плагины располагаются в папке ресурсов сервера, в **`Resources/Server`**, каждый в собственной
подпапке:

```
Resources/Server/
├── MyPlugin/
│   ├── main.lua
│   └── lua/
│       └── helper.lua
└── CobaltEssentials/
    └── ...
```

По умолчанию папка пуста. Чтобы установить плагин, просто положите его папку сюда (точно так
же, как в BeamMP). О том, как это соотносится с клиентскими модами, см.
[Ресурсы и моды](/ru/hosting/resources/).

## Одна папка плагина — одно Lua-состояние

Основная модель — идентичная BeamMP — такова:

- **Каждая папка плагина — это одно Lua-состояние, работающее в собственном потоке.** Плагины
  изолированы друг от друга; они взаимодействуют через события, а не через общие глобальные
  переменные.
- **Все файлы `.lua` верхнего уровня в папке загружаются в это единое общее состояние** в
  лексикографическом порядке. Именно поэтому прилагаемые примеры названы `00_framework.lua`,
  `10_admin.lua`, `20_dimensions.lua`, … — числовые префиксы задают порядок загрузки, и более
  поздние файлы могут использовать то, что определили более ранние (например, `10_admin`
  регистрирует реестр команд, в который `20_dimensions` добавляет свои).
- **Вложенные модули загружаются через `require`.** И `<plugin>/?.lua`, и `<plugin>/lua/?.lua`
  находятся в `package.path`, поэтому `require("helper")` находит `lua/helper.lua`.

:::note
Файлы в **одной и той же** папке разделяют состояние и могут напрямую вызывать друг друга.
Файлы в **разных** папках плагинов — нет, поэтому держите самодостаточную функциональность в
одной папке.
:::

## Жизненный цикл и события

Плагин управляется событиями. При загрузке его код верхнего уровня выполняется один раз
(регистрация обработчиков, инициализация состояния); после этого сервер вызывает ваши
обработчики по мере наступления событий. Ключевые события жизненного цикла:

| Событие | Когда |
|---|---|
| `onInit` | Lua-состояние запущено. |
| `onPlayerJoin` | Игрок завершил авторизацию и синхронизацию. |
| `onPlayerDisconnect` | Игрок покинул сервер. |
| `onChatMessage` | Игрок отправил сообщение в чат (можно наложить вето). |
| `onVehicleSpawn` / `postVehicleSpawn` | Игрок спавнит автомобиль (на первое можно наложить вето). |
| `onVehicleDeleted`, `onVehicleEdited`, `onVehicleReset` | Жизненный цикл автомобиля. |
| `onShutdown` | Сервер останавливается. |

На некоторые события можно наложить **вето**, вернув ненулевое целое число (например, возврат
`1` из `onChatMessage` поглощает сообщение — именно так пример плагина администрирования не
пускает `/commands` в публичный чат). Полный список событий, их аргументов и того, на какие
можно наложить вето, приведён в [Lua API сервера](/ru/plugins/server-api/).

## Два API, оба доступны

Внутри плагина вам доступны:

- **Совместимый с BeamMP глобальный API** — `MP.*`, `Util.*`, `FS.*`, `Http.*`, с моделью
  событий BeamMP (`MP.RegisterEvent`, `CreateEventTimer`, вето через `return 1`). Существующие
  плагины BeamMP используют его и продолжают работать.
- **Пространственно именованный API `NodeMP.*`** — более чистый интерфейс поверх тех же
  примитивов (`NodeMP.events.on`, `NodeMP.players.get`, `NodeMP.chat.broadcast`,
  `NodeMP.vehicles.remove`, …). Для нового кода предпочтительнее он.

Минимальный плагин (`Resources/Server/Hello/main.lua`):

```lua
local NodeMP = _G.NodeMP

NodeMP.events.on("onPlayerJoin", function(playerId)
    NodeMP.chat.send(playerId, "Welcome to the server!")
end)
```

## Учитесь на примерах

В репозитории поставляются пять примеров плагинов в `server/examples/server-plugins`, которые
вместе демонстрируют большую часть API:

- **`00_framework.lua`** — реестр модулей + клиентский манифест (включение/выключение функций,
  сохраняемая конфигурация).
- **`10_admin.lua`** — команды чата через `/` с моделью прав и переиспользуемым реестром
  команд.
- **`20_dimensions.lua`** — параллельные миры на одной карте (модуль фреймворка, по умолчанию
  выключен).
- **`30_persistence.lua`** — сохранение и восстановление мира автомобилей на диск.
- **`40_welcome.lua`** — приветствие при подключении / MOTD.

Скопируйте папку в `Resources/Server`, чтобы попробовать.
