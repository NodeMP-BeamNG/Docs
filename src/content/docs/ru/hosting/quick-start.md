description: Запустите игровой сервер NodeMP на своей машине и подключитесь к нему по локальной петле (loopback) — без бэкенда и без аккаунтов.
---

Это руководство поможет запустить **игровой сервер** NodeMP на вашей собственной машине и
подключиться к нему локально. Подключение по локальной петле (loopback, `127.0.0.1`)
доверяется автоматически, поэтому для проверки на одном ПК вам **не нужны ни бэкенд, ни
аккаунт** — для игроков с той же машины сервер пропускает шаг погашения билета.

Когда позже вы захотите сделать сервер доступным для других людей или добавить его в
публичный браузер серверов, см. [Запуск сервера](/ru/hosting/running/) и
[Регистрация хоста](/ru/hosting/registering/).

## Что вам понадобится

- **Бинарный файл сервера** NodeMP (`NodeMP-Server` в Linux, `NodeMP-Server.exe` в Windows).
  Пока не опубликованы готовые сборки, вы собираете его из исходников с помощью прилагаемого
  Dockerfile — команды сборки см. в разделе [Запуск сервера](/ru/hosting/running/).
- **BeamNG.drive**, а также лаунчер и мод NodeMP на том же ПК, чтобы вы могли подключиться.
  См. [Установка лаунчера](/ru/players/install/).
- Один свободный порт: по умолчанию **30814** (TCP **и** UDP).

## 1. Поместите бинарный файл в отдельную папку

Сервер записывает свою конфигурацию и создаёт папки `Resources/` **рядом с рабочим
каталогом, из которого он запущен**, поэтому выделите ему собственную папку — например,
`C:\NodeMP` в Windows или `/opt/nodemp` в Linux.

## 2. Первый запуск создаёт конфигурацию

Запустите сервер один раз из этой папки:

```bash
# Linux
./NodeMP-Server
```

```powershell
# Windows (PowerShell)
.\NodeMP-Server.exe
```

При первом запуске файла `ServerConfig.toml` ещё нет, поэтому сервер создаёт новый с
разумными значениями по умолчанию, создаёт пустые папки `Resources/Server` и
`Resources/Client`, а затем продолжает работать с этими настройками. В консоли вы увидите
строку вроде *«No config found … Writing a fresh one.»*.

:::tip
В Windows, если вы собирали из исходников, вспомогательный скрипт `run-server-local.bat`
сделает это за вас: он запускает свежесобранный `build\Release\NodeMP-Server.exe` из рабочей
папки `run-local\` и оставляет окно открытым, чтобы вы могли читать журнал.
:::

## 3. Минимальный `ServerConfig.toml`

Сгенерированные значения по умолчанию уже подходят для локальной проверки. Здесь важны лишь
следующие значения:

```toml
[Server]
Name    = "My NodeMP Server"
Private = true            # keep it off the public browser while testing

[Network]
IP            = "0.0.0.0" # listen on all interfaces
Port          = 30814     # TCP + UDP
TrustLoopback = true      # trust 127.0.0.1 peers with no backend auth (local dev)

[Backend]
HostId     = ""           # leave blank for a private / LAN-only server
HostSecret = ""
```

Именно `TrustLoopback = true` позволяет подключаться к собственному серверу без бэкенда.
Пока оставьте учётные данные в `[Backend]` пустыми — они нужны только для появления в
публичном списке. Описание каждого ключа см. в разделе
[Конфигурация](/ru/hosting/configuration/).

:::note
Измените значение, **сохраните файл, затем перезапустите сервер**, чтобы применить его.
Настройки читаются один раз при запуске.
:::

## 4. Запустите сервер

Снова запустите сервер из его папки. При успешном запуске выводятся имя узла, карта и сводка
по аплинку (порт, максимум игроков, максимум автомобилей), а сеть отмечается как *private /
LAN*, поскольку учётные данные не заданы. Оставьте окно открытым.

## 5. Подключитесь из BeamNG

Когда лаунчер запущен и вы вошли в аккаунт (или как гость), откройте **NodeMP Multiplayer**
в BeamNG и подключитесь по прямому IP к:

```
127.0.0.1:30814
```

Поскольку это подключение по локальной петле, сервер доверяет ему, не обращаясь к бэкенду.
Готово — заспавните автомобиль, и вы уже едете на собственном сервере.
