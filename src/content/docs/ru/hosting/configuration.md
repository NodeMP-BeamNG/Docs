description: Каждый ключ в ServerConfig.toml — секции [Server], [Network], [Backend], [Gameplay], [Logging] и [Updates] — с его типом, значением по умолчанию и назначением.
---

Сервер NodeMP настраивается через один файл — **`ServerConfig.toml`** — в папке, из которой
вы запускаете сервер. Он создаётся автоматически при первом запуске (см.
[быстрый старт](/ru/hosting/quick-start/)) с разумными значениями по умолчанию, а дальше вы
редактируете его сами.

:::note
Настройки читаются **один раз при запуске**. Измените значение, сохраните файл, затем
**перезапустите сервер**, чтобы применить его. Оставляйте кавычки вокруг текстовых значений;
числа и `true`/`false` остаются без кавычек.
:::

## `[Server]` — идентификация

Как ваш узел представляется в браузере серверов и кто может подключаться.

| Ключ | Тип | По умолчанию | Назначение |
|---|---|---|---|
| `Name` | string | `"NodeMP Server"` | Имя, отображаемое в браузере серверов. |
| `Description` | string | `"Default NodeMP server"` | Однострочное описание в браузере. |
| `Tags` | string | `""` | Теги через запятую, например `"Freeroam,Drift"`. |
| `Map` | string | `"/levels/gridmap_v2/info.json"` | Уровень (карта), который размещает сервер. |
| `ResourceFolder` | string | `"Resources"` | Папка с серверными плагинами и клиентскими модами. См. [Ресурсы и моды](/ru/hosting/resources/). |
| `Private` | bool | `true` | Скрыть узел из публичного браузера. Установите `false`, чтобы попасть в список (также требуются учётные данные бэкенда и достижимый порт). |
| `AllowGuests` | bool | `true` | Разрешить подключение игрокам без аккаунта. |

## `[Network]` — аплинк

Прослушивающий сокет и ограничения на сессию.

| Ключ | Тип | По умолчанию | Назначение |
|---|---|---|---|
| `IP` | string | `"0.0.0.0"` | Адрес привязки; `0.0.0.0` означает все интерфейсы. |
| `Port` | int | `30814` | Прослушиваемый порт — используется **и** для TCP, **и** для UDP. |
| `MaxPlayers` | int | `10` | Максимум одновременных игроков. |
| `MaxCars` | int | `5` | Сколько автомобилей может заспавнить каждый игрок. |
| `TrustLoopback` | bool | `true` | Доверять пирам `127.0.0.1` без авторизации через бэкенд (локальная разработка). **Установите `false`, если сервер находится за обратным прокси**, который выглядит как loopback, иначе внешние игроки могут быть восприняты как доверенные. |

## `[Backend]` — учётные данные сети

Учётные данные, которые сервер предъявляет бэкенду, чтобы открыть свою хост-сессию и
появиться в публичном списке. Оставьте оба поля пустыми, чтобы остаться **приватным / только
для локальной сети (LAN)**.

| Ключ | Тип | По умолчанию | Назначение |
|---|---|---|---|
| `HostId` | string | `""` | Идентификатор хоста, выданный бэкендом. |
| `HostSecret` | string | `""` | Секрет хоста, выданный бэкендом (показывается лишь один раз). |

Получите их, зарегистрировав свой хост — см. [Регистрация хоста](/ru/hosting/registering/);
эта процедура также может заполнить данную секцию за вас.

:::caution
**Адрес** самого бэкенда здесь **не** задаётся. Он берётся из переменной окружения
`NODEMP_BACKEND_URL` (по умолчанию `https://api.nodemp.com`). См.
[Запуск сервера](/ru/hosting/running/).
:::

## `[Gameplay]` — набор правил

| Ключ | Тип | По умолчанию | Назначение |
|---|---|---|---|
| `InformationPacket` | bool | `false` | Для продвинутых; оставьте `false`, если не указано иное. |

## `[Logging]` — телеметрия

| Ключ | Тип | По умолчанию | Назначение |
|---|---|---|---|
| `LogChat` | bool | `true` | Дублировать сообщения чата в консоль. |
| `Debug` | bool | `false` | Подробное трассировочное логирование и логирование событий. Удобно при тестировании плагинов. |

## `[Updates]`

Управляет напоминанием «ваш сервер устарел».

| Ключ | Тип | По умолчанию | Назначение |
|---|---|---|---|
| `ImScaredOfUpdates` | bool | `false` | Установите `true`, чтобы отключить напоминание об устаревании. |
| `UpdateReminderTime` | int | `360` | Интервал между напоминаниями об обновлении в минутах. |

## `[General]` — зеркало совместимости с BeamMP

При генерации конфигурации NodeMP также добавляет секцию `[General]` в плоском формате BeamMP
(`Name`, `Port`, `MaxCars`, `MaxPlayers`, `Map`, `Private`, `Description`, `Tags`, `Debug`,
`LogChat`, `ResourceFolder`, `AuthKey`). Многие серверные плагины BeamMP читают
`ServerConfig.toml` напрямую (низкоуровневый файловый ввод-вывод в обход API) и ожидают именно
эту секцию.

Сам NodeMP **игнорирует** `[General]` — он читает собственные типизированные секции выше.
Относитесь к `[General]` как к зеркалу, а не как к источнику истины:

- Изменяйте реальные значения в `[Server]` / `[Network]` и т. д.
- Если вы вручную правите значение, которое плагин читает из `[General]`, поддерживайте оба в
  синхронизации.
- Для конфигурации, созданной более старой версией, где нет `[General]`, добавьте её вручную
  или удалите `ServerConfig.toml` и дайте серверу сгенерировать новую.

Полную картину см. в [Совместимости с BeamMP](/ru/introduction/beammp-compatibility/).

## Пример

Полностью сгенерированный `ServerConfig.toml` (комментарии убраны):

```toml
[Server]
Name           = "NodeMP Server"
Description    = "Default NodeMP server"
Tags           = ""
Map            = "/levels/gridmap_v2/info.json"
ResourceFolder = "Resources"
Private        = true
AllowGuests    = true

[Network]
IP            = "0.0.0.0"
Port          = 30814
MaxPlayers    = 10
MaxCars       = 5
TrustLoopback = true

[Backend]
HostId     = ""
HostSecret = ""

[Gameplay]
InformationPacket = false

[Logging]
LogChat = true
Debug   = false

[Updates]
ImScaredOfUpdates  = false
UpdateReminderTime = 360
```
