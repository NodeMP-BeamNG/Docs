Быстрый справочник по терминам, используемым в этой документации. О том, как составные
части работают вместе, см. [Что такое NodeMP](/ru/introduction/what-is-nodemp/).

| Термин | Определение |
|---|---|
| **Лаунчер** | Десктопное приложение, которое вы запускаете вместе с BeamNG; оно выполняет ваш вход, поддерживает внутриигровой мод в актуальном состоянии и проксирует ваше подключение к игровым серверам. |
| **Мод** | Внутриигровое дополнение BeamNG (Lua с интерфейсом на Vue/Angular), которое добавляет экран мультиплеера, чат, список игроков, таблички с именами и синхронизацию автомобилей. |
| **Хост** | Зарегистрированный игровой сервер — его владелец и машина — который может появляться в публичном списке серверов. |
| **Бэкенд** | Центральный API сети: аккаунты и вход, билеты на подключение, публичный список серверов, регистрация хостов и метаданные релизов. |
| **Владелец синхронизации** | Единственный подключённый клиент, который в данный момент отвечает за синхронизацию конкретного автомобиля; сервер переназначает эту роль, если такой клиент отключается. |
| **Билет на подключение** | Одноразовый токен, который лаунчер получает от бэкенда и передаёт серверу, чтобы подтвердить, что вам разрешено подключиться. |
| **Секрет хоста** | Секретные учётные данные, которые сервер предъявляет (вместе со своим host id), чтобы открыть аутентифицированную сессию с бэкендом. |
| **Измерение** | Параллельный мир на той же карте; игроки в разных измерениях не видят друг друга и не сталкиваются. По умолчанию отключено. |
| **Присутствие** | Актуальная запись «онлайн» хоста, на которой основан публичный браузер серверов; поддерживается свежей за счёт маяка и истекает после короткого времени жизни (TTL). |
| **Маяк** | Периодическое обновление, которое работающий сервер отправляет бэкенду, чтобы оставаться в списке браузера серверов. |
